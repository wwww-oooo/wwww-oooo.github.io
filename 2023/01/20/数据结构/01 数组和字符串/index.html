01 数组和字符串

  1. 理解 数组 的基本概念及其操作方式；
  2. 理解 二维数组 的基本概念，熟悉二维数组的使用；
  3. 了解 字符串 的概念以及字符串所具有的不同特性；
  4. 理解 字符串匹配中的 KMP 算法；
  5. 能够运用 双指针 解决实际问题。

1 数组简介

1.1 集合、列表和数组

  • 集合
    • 集合里的元素类型不一定相同
    • 集合里的元素没有顺序
    • 集合并 不直接存在 于编程语言
  • 列表（线性列表）
    • 按照一定 线性顺序 排列的有限序列
    • 具有顺序、长度可变
    • 编程语言中最常见的形式：数组、链表、栈、队列
    • 没有索引
    • 列表中的元素在内存中可能彼此相邻，也可能不相邻
  • 数组
    • 将 相同类型 的元素存储于 连续内存空间 的数据结构，其 长度不可变
    • 用 索引 表示数据在数组中的位置
    • 数组中的元素在内存中是 连续存储 的，且每个元素占用==相同大小==的内存
  • 可变数组：基于数组和扩容机制实现，相比普通数组更加灵活
    • Java：集合类 List<Integer> array = new ArrayList<>();

1.2 数组的操作

读取元素

  • 通过访问索引的方式读取数组中的元素
  • 对于数组，计算机会在内存中为其申请一段 连续 的空间，并且会记下索引为 0 处的内存地址
  • 读取元素的时间复杂度：\(O(1)\)

查找元素

  • 从数组头部逐步向后查找，找到停止，否则搜索到数组末尾
  • 查找元素的时间复杂度为 \(O(N)\)

插入元素

  1. 插入到数组末尾
    • 一步，计算需要插入的内存地址，插入数据
  2. 插入到数组中的其他位置
    1. 为该元素所要插入的位置 腾出 空间
    2. 进行插入操作
    • 链表 可以有效解决频繁插入操作造成的时间浪费问题

删除元素

  1. 删除数组中的某个元素后，留下 空缺 的位置
  2. 数组中的元素在内存中是 连续 的，这就使得后面的元素需对该位置进行 填补 操作。
  3. 删除操作具有线性时间复杂度，即时间复杂度为 \(O(N)\)

2 二维数组

  • 二维数组：数组中的每个元素是一维数组
  • 二维数组的本质上仍然是一个一维数组，内部的一维数组仍然从索引 0 开始、
  • 可以将它看作一个矩阵，并 处理矩阵的相关问题
    • 矩阵旋转
    • 对角线遍历
    • 对子矩阵的操作

3 字符串

3.1 为何单独讨论字符串类型？

  1. 字符串的基本操作对象通常是 字符串整体或者其子串
    • 理解：字符串里的单个下标对应的值可能没有任何意义，而是其某个字串构成的单词由意义
  2. 字符串的操作比其他数据类型更复杂

3.2 字符串的比较

  • 什么语言能够用 == 比较两个字符串？
    • 判断规则：该语言是否支持 运算符重载
    • 【支持】C++、Python：可以使用 == 比较两个字符串
    • 【不支持】Java：不可以使用 == 比较两个字符串，会判断这两个对象是否是同一个对象

3.3 字符串连接

  • 对于不同的编程语言中，字符串可能是可变的，也可能是不可变的
    • 不可变 意味着 一旦字符串被初始化，你就无法改变它的内容
    • 可变：C++
    • 不可变：Java、Python
  • 在 字符串不可变 的语言中，哪怕你只是想修改其中的一个字符，也 必须 创建一个新的字符串
  • 对于 Java来说，由于字符串是不可变的，因此在连接时首先为新字符串分配足够的空间，复制旧字符串中的内容并附加到新字符串
    • 总时间复杂度：\(O(N^2)\)
    • 解决办法
      • 希望字符串是 可变 的，则可以使用 toCharArray 将其转换为 字符数组
      • 经常必须 连接字符串，最好使用一些其他的数据结构，如 StringBuilder

3.4 字符串匹配算法 KMP

KMP 算法

  • Knuth–Morris–Pratt（KMP）算法是一种改进的字符串匹配算法，它的核心是利用匹配失败后的信息，尽量减少模式串与主串的匹配次数以达到快速匹配的目的
  • 时间复杂度是 \(O(m+n)\)

【使用双指针的典型场景】 - 从两端向中间迭代数组 - 一个指针从头部开始，一个指针从尾部开始 - 快慢指针（同向运动） - 滑动窗口
